# 2026-03-24 Two Map Sequence Demo

## 1. 목적

중간보고서 시연 단계에서는 하나의 맵만 로드하는 것이 아니라, 준비된 여러 맵을 순서대로 교체하면서 차량을 다시 소환하고 반복적으로 주행할 수 있어야 한다.

이번 작업에서는 큰 경기장형 트랙 대신 `작고 열린 형태의 코스 2개`를 미리 생성하고, 버튼 입력으로 다음 맵을 로드하는 데모 프로그램을 구성하였다.

## 2. 구현 파일

- `scripts/carla_runner/run_manual_map_sequence_demo.py`

## 3. 준비된 맵 구성

프로그램 시작 시 두 개의 OpenDRIVE 맵을 미리 생성한다.

- 첫 번째 맵: `..._case_001`
  - 짧은 직선 구간 뒤 완만한 좌회전을 지나 도착점으로 이어지는 작은 오픈 코스
- 두 번째 맵: `..._case_002`
  - 짧은 직선과 S자형 곡선을 거쳐 도착점으로 이어지는 작은 오픈 코스

두 맵 모두 폐곡선이 아니라 `시작 지점과 끝 지점이 분리된 형태`로 설계하였다.

또한 각 맵은 자체적인 spawn point 정보를 함께 가진다. 따라서 차량은 CARLA가 임의로 제공하는 기본 spawn point가 아니라, 해당 맵에 대해 미리 정의된 시작 지점 근처에서 소환된다.

## 4. 동작 방식

1. 첫 번째 맵을 CARLA에 로드한다.
2. 차량을 맵 위에 스폰한다.
3. 사용자가 수동으로 주행한다.
4. `N` 키를 누르면 다음 준비된 맵을 로드한다.
5. 차량을 다시 소환한다.
6. 두 번째 맵에서도 다시 주행한다.
7. 다시 `N` 키를 누르면 첫 번째 맵으로 순환할 수 있다.

## 5. 조작 키

- `W / A / S / D`: 수동 주행
- `Space`: 핸드브레이크
- `R`: 현재 맵 다시 로드 + 차량 재소환
- `N`: 다음 준비된 맵 로드 + 차량 재소환
- `ESC`: 종료

## 6. 실행 명령

```powershell
cd C:\Users\user\Documents\Codex_Workspace\AutoDriveEnvironmentDesigner
py -3.7 scripts\carla_runner\run_manual_map_sequence_demo.py --pythonapi-path C:\Users\user\Documents\CARLA_0.9.15\WindowsNoEditor --map-id sequence_demo_live
```

## 7. 의미

이번 단계로 다음 반복 구조를 시연할 수 있게 되었다.

`맵 1 로드 -> 차량 소환 -> 주행 -> 맵 2 로드 -> 차량 소환 -> 주행 -> 다시 맵 1 로드`

따라서 중간보고서에서는 “단일 맵 검증”을 넘어 “복수의 맵을 순차적으로 교체하며 반복 실험이 가능한 구조”를 구현하였다고 설명할 수 있다.
