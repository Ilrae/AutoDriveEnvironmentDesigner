# 2026-03-24 Manual Map Reload Cycle

## 1. 목적

중간보고서 단계에서 필요한 기능은 다음과 같이 정리하였다.

1. 새로운 OpenDRIVE 맵을 생성한다.
2. CARLA에 해당 맵을 로드한다.
3. 차량을 맵 위에 다시 소환한다.
4. 사용자가 수동으로 다시 주행한다.

즉, 한 번의 데모가 아니라 `맵 교체 -> 차량 재소환 -> 재주행`이 반복 가능한 구조를 만드는 것이 이번 작업의 목적이다.

## 2. 구현 파일

- `scripts/carla_runner/run_manual_track_demo.py`

## 3. 핵심 변경 사항

### 3-1. 맵 시퀀스 개념 추가

수동 주행 데모 스크립트에서 맵을 한 번만 로드하는 것이 아니라, 순번이 붙은 시퀀스 형태로 맵을 생성하도록 변경하였다.

- 예: `manual_track_demo_live_case_001`
- 예: `manual_track_demo_live_case_002`

새로운 맵을 요청할 때마다 새로운 map id를 만들고, 새로운 `.xodr`와 manifest를 생성한다.

### 3-2. 현재 맵 재시작 기능 추가

수동 주행 도중 현재 맵을 다시 시작하고 싶을 때 다음 동작을 수행하도록 구현하였다.

- 현재 차량 제거
- 현재 카메라 제거
- 현재 맵 설정 유지
- 같은 맵을 다시 로드
- 차량 재소환

키:

- `R`: 현재 맵 다시 로드 및 차량 재소환

### 3-3. 다음 맵 로드 기능 추가

수동 주행 도중 다음 맵으로 넘어갈 수 있도록 다음 동작을 구현하였다.

- 다음 시퀀스 번호 계산
- 다음 map id 생성
- 난이도(level) 증가
- 필요 시 좌/우 방향 번갈아 생성
- 새 `.xodr` 생성
- CARLA에 새 맵 로드
- 차량 재소환

키:

- `N`: 다음 맵 생성, 로드, 차량 재소환

### 3-4. 트랙 파라미터 자동 변화

맵이 완전히 동일하게 반복되지 않도록 시퀀스가 증가할 때 트랙 파라미터가 조금씩 바뀌도록 설정하였다.

- lane width 감소
- straight length 감소
- track radius 감소
- level 증가
- 곡선 방향 좌/우 번갈아 적용 가능

이렇게 하여 중간보고서에서 “하나의 맵만 로드하는 단계”를 넘어 “여러 맵을 순차적으로 생성하고 로드하는 단계”까지 진행되었음을 설명할 수 있다.

## 4. 사용자 조작 방식

현재 기준 조작 키는 다음과 같다.

- `W / A / S / D` 또는 방향키: 차량 수동 조작
- `Space`: 핸드브레이크
- `R`: 현재 맵 다시 로드 + 차량 재소환
- `N`: 다음 맵 생성 + 로드 + 차량 재소환
- `ESC`: 종료

## 5. 실행 명령

```powershell
cd C:\Users\user\Documents\Codex_Workspace\AutoDriveEnvironmentDesigner
py -3.7 scripts\carla_runner\run_manual_track_demo.py --pythonapi-path C:\Users\user\Documents\CARLA_0.9.15\WindowsNoEditor --map-id manual_track_demo_live --level 0.30 --curve-direction left --keep-vehicle
```

## 6. 검증 내용

다음 항목을 확인하였다.

- 초기 맵 로드 성공
- 초기 차량 스폰 성공
- 수동 주행 창 유지 성공
- 코드 레벨에서 맵 2회 연속 로드 성공
- 두 번째 맵에서도 차량 재소환 성공

## 7. 중간보고서에 넣을 수 있는 내용

이번 단계에서 시스템은 다음 순환 구조를 수행할 수 있게 되었다.

`맵 생성 -> CARLA 로드 -> 차량 소환 -> 수동 주행 -> 다음 맵 생성 -> 다시 로드 -> 차량 재소환`

따라서 중간보고서에서는 단일 맵 검증 수준을 넘어, 반복적인 환경 교체와 재실험이 가능한 구조를 구현하였다고 정리할 수 있다.
