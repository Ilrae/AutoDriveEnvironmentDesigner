# 차량 및 Autopilot 사용 방식 정리

## 1. 문서 목적

이 문서는 본 프로젝트에서 차량과 주행 제어를 어떤 방식으로 사용했는지 정리하기 위한 보조 문서이다.

특히 중요한 점은, 본 프로젝트의 핵심이 `자율주행 제어 알고리즘 구현`이 아니라 `OpenDRIVE 기반 도로 환경 생성과 그 결과 검증`이라는 사실을 명확히 하는 것이다.

## 2. 프로젝트 범위에서의 차량 역할

본 프로젝트에서 차량은 `연구 대상 그 자체`라기보다, 생성한 도로 환경이 실제로 주행 가능한지 확인하기 위한 `검증 수단`으로 사용된다.

즉, 차량의 역할은 아래와 같다.

- 생성한 OpenDRIVE 맵 위에 실제 차량을 올릴 수 있는지 확인
- 해당 맵에서 기본적인 주행이 가능한지 확인
- 도로 형상 변경에 따라 주행 가능 여부를 빠르게 검증

따라서 본 프로젝트의 핵심은 차량 제어기가 아니라 다음 요소들에 있다.

- OpenDRIVE 맵 생성
- CARLA 맵 로딩
- 주행 결과 수집
- 난이도 조절 및 환경 생성 로직

## 3. 사용한 방식

이번 프로젝트의 테스트 단계에서는 `CARLA에 기본 포함된 차량 blueprint`와 `CARLA 내장 autopilot 기능`을 사용하였다.

즉, 별도의 자율주행 알고리즘이나 사용자 정의 제어기를 구현하지 않았다.

사용 목적은 다음과 같다.

- 생성된 맵 위에서 차량이 실제로 spawn 되는지 확인
- CARLA가 제공하는 기본 주행 기능으로 도로를 따라 움직일 수 있는지 확인
- 맵 생성 시스템 검증에 집중하기 위해 차량 제어 구현 부담을 줄이기

## 4. 왜 내장 autopilot을 사용했는가

내장 autopilot을 사용한 이유는 다음과 같다.

1. 본 프로젝트의 핵심은 자율주행 제어기가 아니라 맵 생성 시스템이기 때문이다.
2. 초기 단계에서 차량 제어 로직까지 직접 구현하면 프로젝트 범위가 불필요하게 커지기 때문이다.
3. CARLA 기본 기능만으로도 생성한 트랙이 주행 가능한지 빠르게 검증할 수 있기 때문이다.

즉, 내장 autopilot은 본 프로젝트에서 `평가 대상`이 아니라 `검증 도구`로 사용되었다.

## 5. 보고서에 넣을 수 있는 설명 문장

### 짧은 설명

본 프로젝트에서는 자율주행 제어 알고리즘 자체를 구현하지 않고, CARLA에서 기본 제공하는 차량 blueprint와 autopilot 기능을 사용하여 생성된 OpenDRIVE 맵의 주행 가능 여부를 검증하였다.

### 조금 더 자세한 설명

차량 제어 로직은 본 연구의 핵심 범위가 아니므로, 초기 검증 단계에서는 CARLA에 기본 포함된 차량과 autopilot 기능을 활용하였다. 이를 통해 생성된 OpenDRIVE 기반 도로 환경이 실제로 로드되고 주행 가능한지를 우선 확인하였으며, 프로젝트의 핵심 개발 범위는 도로 환경 생성과 실험 자동화 구조에 두었다.

### 범위 구분 강조용 설명

본 프로젝트는 자율주행 알고리즘 개발이 아니라, 자율주행 실험에 사용할 도로 환경을 자동 생성하는 시스템을 목표로 한다. 따라서 차량 주행은 CARLA 기본 기능을 이용한 검증 단계로 처리하였고, 연구의 중심은 OpenDRIVE 생성과 환경 난이도 조절 로직에 두었다.

## 6. 현재 구현과 연결되는 스크립트

- `scripts/carla_runner/test_spawn_vehicle.py`
- `scripts/carla_runner/run_first_validation.py`
- `scripts/carla_runner/vehicle_utils.py`

이 스크립트들은 CARLA에 기본 포함된 차량을 선택하고, autopilot을 활성화하여 생성된 맵 위에서 짧은 검증 주행을 수행하는 역할을 한다.

## 7. 연속 데모 주행에 대한 보조 설명

초기 검증 단계에서는 CARLA 내장 autopilot을 사용해 생성된 맵이 주행 가능한지 확인하였다.

다만 standalone OpenDRIVE 기반의 커스텀 경기장 트랙에서는 내장 autopilot만으로 장시간 안정적인 시연 화면을 유지하기 어려운 경우가 있었다. 그래서 시연용 보조 도구로 `scripts/carla_runner/run_stadium_track_demo.py`를 추가했고, 이 스크립트에서는 필요 시 가벼운 waypoint loop controller를 사용해 차량이 트랙을 따라 천천히 계속 주행하도록 구성하였다.

즉, 보고서에서는 아래처럼 구분하는 것이 자연스럽다.

- 기본 주행 가능 여부 검증: CARLA 기본 차량 + CARLA 내장 autopilot
- 장시간 시연용 보조 데모: 가벼운 waypoint loop controller 사용

## 8. 정리

현재 단계에서 차량과 autopilot은 본 프로젝트의 핵심 산출물이 아니라, `생성한 맵이 실제로 사용 가능한지 확인하기 위한 보조 검증 구성요소`로 정리하는 것이 가장 자연스럽다.

따라서 보고서에서는 아래처럼 구분해서 쓰는 것이 적절하다.

- 핵심 구현: OpenDRIVE 맵 생성, CARLA 로딩, 실험 자동화
- 보조 검증 수단: CARLA 기본 차량 및 기본 autopilot 사용
