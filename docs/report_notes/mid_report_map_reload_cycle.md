# 중간보고서용 맵 재로딩 사이클 정리

## 1. 기능 요약

중간보고서 단계에서 구현한 핵심 기능은 다음과 같다.

- OpenDRIVE 기반 경기장형 트랙 생성
- CARLA에서 생성된 트랙 로드
- 차량 스폰
- 수동 주행 검증
- 새로운 맵 생성 후 재로딩
- 새로운 맵에서 차량 재소환 후 다시 주행

즉, 시스템은 단일 맵을 한 번 띄우는 수준이 아니라, `맵 변경 -> 차량 재배치 -> 재주행`까지 가능한 반복 구조를 갖추게 되었다.

## 2. 보고서용 설명 문장

본 프로젝트는 OpenDRIVE 기반 주행 코스를 자동 생성하고, 해당 코스를 CARLA에서 반복적으로 검증할 수 있는 환경을 구축하는 것을 목표로 한다. 중간보고서 단계에서는 생성된 맵을 CARLA에 로드한 뒤 차량을 소환하여 수동 주행으로 검증할 수 있도록 구성하였으며, 추가적으로 새로운 맵을 다시 생성하여 CARLA에 재로딩하고 차량을 다시 소환한 뒤 연속적으로 실험할 수 있는 사이클을 구현하였다.

## 3. 수동 주행을 선택한 이유

초기에는 CARLA 기본 autopilot을 활용하여 생성된 트랙 위에서 자동 주행 검증을 수행하려고 하였다. 그러나 custom OpenDRIVE 기반 트랙에서는 CARLA 기본 autopilot의 주행 안정성이 충분하지 않았고, 이는 본 프로젝트의 핵심 목표인 맵 생성 및 평가 구조 구현과는 거리가 있었다. 따라서 중간보고서 단계에서는 차량 제어 알고리즘 개발을 프로젝트 범위에서 제외하고, CARLA 기본 수동 주행 기능을 이용하여 생성된 트랙의 주행 가능성을 검증하는 방식으로 방향을 조정하였다.

## 4. 현재 데모에서 가능한 동작

- 첫 번째 맵 생성 및 로딩
- 차량 스폰
- 수동 주행
- 현재 맵 재시작
- 다음 맵 생성 및 재로딩
- 다음 맵에서 차량 재소환
- 반복 주행

## 5. 구현 파일

- `scripts/carla_runner/run_manual_track_demo.py`

## 6. 발표/시연 시 강조할 점

- 프로젝트의 핵심은 자율주행 알고리즘 개발이 아니라, 주행 환경 생성과 평가 구조 설계이다.
- 차량 주행은 검증 수단이며, 현재 단계에서는 수동 주행을 사용하였다.
- 시스템은 한 번의 맵 로딩으로 끝나지 않고, 새로운 맵을 다시 생성하고 교체하는 반복 실험 구조를 갖는다.
