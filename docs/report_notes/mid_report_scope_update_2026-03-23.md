# 중간보고서 범위 조정 메모

## 1. 조정 배경

초기에는 생성한 트랙 위에서 차량을 CARLA 기본 autopilot 또는 간단한 자율주행 로직으로 주행시키는 방향을 검토하였다. 그러나 standalone OpenDRIVE 기반 custom 트랙에서는 CARLA 기본 autopilot의 주행 품질이 충분히 안정적이지 않았고, 차량 제어 성능을 높이기 위한 추가 구현은 프로젝트 본래 목표와 거리가 있었다.

## 2. 조정 내용

중간보고서 단계에서는 차량 주행 방식을 다음과 같이 정리한다.

- 자율주행 알고리즘 구현: 본 프로젝트 핵심 범위에서 제외
- 차량 주행 방식: CARLA 기본 수동 주행 기능을 이용한 검증
- autopilot 사용: 초기 smoke test 또는 보조 확인 용도

즉, 주행 제어를 연구하는 것이 아니라 생성한 맵이 실제로 주행 가능한지, 그리고 해당 코스가 얼마나 쉬운지 또는 어려운지를 평가하는 방향으로 범위를 명확히 한다.

## 3. 중간보고서에 쓸 수 있는 문장

초기에는 CARLA의 기본 autopilot 기능을 활용하여 생성한 트랙 위에서 자동 주행 검증을 수행하고자 하였다. 그러나 custom OpenDRIVE 트랙 환경에서는 기본 autopilot의 주행 안정성이 충분하지 않아, 중간보고서 단계에서는 차량 제어 알고리즘 구현을 프로젝트 핵심 범위에서 제외하고 CARLA 기본 수동 주행 기능을 활용한 검증 방식으로 방향을 조정하였다. 이에 따라 본 프로젝트는 OpenDRIVE 기반 주행 환경 생성, CARLA 로딩, 코스 검증 및 평가 구조 설계에 초점을 맞추었다.
